# Inventory: where portfolio **value** ($) is shown in the app

Purpose: baseline for tracing inconsistent numbers (e.g. top weekly equal portfolio) across caching vs calculation. This list is **UI surfaces that show a dollar portfolio value or a labeled “Value” column tied to portfolio MTM**, not every mention of the word “portfolio” in marketing copy.

**Convention:** “Platform” = authenticated app under `(platform)/platform/…`. “Non-platform” = public marketing and strategy-model pages under `(public)/…` plus shared components used from both.

---

## Platform

- **`/platform` (Overview)** — `src/components/platform/platform-overview-client.tsx`: spotlight **Portfolio value** (`SpotlightStatCard`), per-tile **Portfolio value** in `OverviewPortfolioTile`, `HoldingsPortfolioValueLine` under the spotlight chart; tiles use `computeOverviewPortfolioValue` + ranked/card state.
- **`/platform/your-portfolios`** — `src/components/platform/your-portfolio-client.tsx`: main **Portfolio value** stat (`SpotlightStatCard`), `HoldingsPortfolioValueLine`, holdings table **Value** column (`formatYourPortfolioCurrency` on live row / weight × notional); sidebar selection can echo the same metrics.
- **`/platform/your-portfolios` (signed-out guest)** — `src/components/platform/your-portfolios-guest-preview.tsx`: blurred **Portfolio value** CTA with **static** `maskSample` copy (not live NAV).
- **Guest local synthetic profile** (`GUEST_LOCAL_PROFILE_ID`, after onboarding hydrate) — still on platform your-portfolios/overview flows: **`fetchGuestPortfolioConfigPerformanceJson`** → **`/api/platform/portfolio-config-performance`**, then **`buildGuestUserEntryPerformancePayload`** (`src/lib/guest-local-profile.ts`) for real rebased series — distinct from the signed-out static preview.
- **`/platform/explore-portfolios`** — `src/components/platform/explore-portfolios-client.tsx`: list **Value** column on each `ConfigCard` (`fmtUsd(cardEndingValue)` with optional `livePoint?.aiPortfolio`); section title **Portfolio values** in chart browse mode; uses `ExplorePortfoliosEquityChart` for multi-line $ curves.
- **`/platform/explore-portfolios` (detail)** — `src/components/platform/explore-portfolio-detail-dialog.tsx`: **Portfolio value (return%)** flip headline, `HoldingsPortfolioValueLine`, movement/holdings rows with **Portfolio value** + `fmtUsd`, `FlipCard` metrics tied to `endingValuePortfolio` / rebased movement.
- **Sidebar portfolio picker (platform shell)** — `src/components/platform/sidebar-portfolio-config-picker.tsx`: per-row **`$10k →` ending $** (`fmtUsd` on `endingValuePortfolio`); merged rank table vs benchmarks; dialog **Portfolio Values** header; browse chart reuses `ExplorePortfoliosEquityChart` (tooltip $ on series).
- **`/platform/performance` (signed-in strategy performance)** — `src/components/platform/performance-page-client.tsx`: **Portfolio value** in Core metrics (`formatCurrency(metrics.endingValue)`); benchmark ending $ in same card grid.
- **Onboarding wizard (`portfolio-onboarding-dialog.tsx`)** — **Portfolio value** step: scaled **`recommendedFm.endingValue`** from **`/api/platform/portfolio-config-performance`** (`formatUsdWhole` / `recommendedNotionalScale`). Shown from onboarding flow (not the `/platform/recommended-portfolio` table page).
- **Explore equity chart (shared)** — `src/components/platform/explore-portfolios-equity-chart.tsx`: chart legend / tooltips for **Portfolio values** and **Portfolio Value at date**; copy references simulated value from $10,000.
- **Holdings line primitive** — `src/components/platform/holdings-portfolio-value-line.tsx`: reusable line **Portfolio value:** + `$` + optional as-of (used from overview, your-portfolios, explore detail, public performance).
- **Performance chart (equity mode)** — `src/components/platform/performance-chart.tsx`: equity view tooltip copy **Portfolio value over time** (Y-axis is $ on equity view).
- **Recommended portfolio metrics** — `src/components/platform/recommended-portfolio-client.tsx`: no headline **Portfolio value** label; shows return/CAGR/Sharpe/drawdown only (included here only to avoid false positives — **no $ portfolio value** on that page’s metric cards).
- **Sign-up / auth “Live results” preview** — `src/components/auth/auth-preview-placeholder.tsx`: **`PerformanceChart`** in **$** (equity mode) from prefetched **`/api/platform/portfolio-config-performance`** per top config + initial **`/api/platform/guest-preview`** payload.

### Client-side cache modules (one list)

These are **application-level** caches (in-memory `Map`, sometimes **sessionStorage**). **Separate** from Next **`unstable_cache`**, and often **separate** from the browser’s own HTTP cache for `Cache-Control: s-maxage=…` on API routes.

| Module | File | What is cached | TTL / lifetime | Invalidation / notes |
|--------|------|----------------|----------------|----------------------|
| **`loadRankedConfigsClient`** | `src/lib/portfolio-configs-ranked-client.ts` | **`portfolio-configs-ranked`** JSON per **`slug`** (`resolved` + `inflight`) | **No TTL** — until **`USER_PORTFOLIO_PROFILES_INVALIDATE_EVENT`** | **`entrySettingsOnly`** → **does not clear** (early return). |
| **`loadUserEntryPayloadCached`** | `src/lib/your-portfolio-data-cache.ts` | **`user-portfolio-performance`** per **`profileId`** | **No TTL** — Map entry until cleared | **`USER_PORTFOLIO_PROFILES_INVALIDATE_EVENT`**: full **`userEntryStore` / `userEntryInflight`** clear, **unless** **`entrySettingsOnly`** + **`profileId`** → delete **that** profile only. Also **`bypassCache`** / **`invalidateUserEntryPerformanceCache`**. |
| **`loadConfigPerfPayloadCached`** | `src/lib/your-portfolio-data-cache.ts` | **`portfolio-config-performance`** per **slug + risk + frequency + weighting** | **No TTL** — Map entry until cleared | **No** listener on `USER_PORTFOLIO_PROFILES_INVALIDATE_EVENT` in this module — entries persist until **`bypassCache`** on the next load or full reload. |
| **`loadExploreEquitySeries`** | `src/lib/explore-equity-series-cache.ts` | **`explore-portfolios-equity-series`** payload per **`slug`** | **~5 min** (`TTL_MS`); memory + **sessionStorage** | **`invalidateExploreEquitySeriesCache`** on profile event unless **`entrySettingsOnly`**. |
| **`loadConfigPerformance`** | `src/lib/portfolio-config-performance-cache.ts` | Normalized **`portfolio-config-performance`** per **`slug` + `configId`** | **~5 min**; memory + **sessionStorage** | `invalidateConfigPerformanceCache` on profile event (when not `entrySettingsOnly`). |
| **Holdings bootstrap** | `src/lib/portfolio-config-holdings-cache.ts` | **`explore-portfolio-config-holdings`** responses | Session + memory layers; optional revalidate | Used for paid holdings prefetch paths. |

**Not** using those modules but still “cached” in the browser: **`PortfolioValuesSection`**, **sidebar picker chart tab**, and some **raw `fetch`** paths rely on **HTTP caching** from the API’s **`Cache-Control`** (e.g. equity-series **`s-maxage=300`**) plus **React state** for the parsed JSON.

### Accuracy notes (double-check summary)

- **`usePublicPortfolioConfigPerformance`** initializes **`rankedConfigs` to `[]`** even when the **[portfolio] RSC** page already called **`getCachedRankedConfigsPayload`** — SSR ranked is used for **validation and `portfolioPageLinks`**, but **not** as hook seed data, so **`#portfolio-values` / top-portfolio CTA** can show loading or **client-cached ranked** until **`loadRankedConfigsClient`** resolves.
- **Overview** does **not** use **`portfolio-config-performance`**; **your-portfolios** uses **both** **`user-portfolio-performance`** and **`portfolio-config-performance`** (plus guest-local path).
- **`/platform/explore-portfolios`** uses **`loadExploreEquitySeries`** (5 min client cache); **`PortfolioValuesSection`** and **sidebar picker chart** use **raw `fetch`** (browser HTTP cache only, **`s-maxage=300`** on the API).
- **Landing Hero + All-portfolios** share cache tag **`landing-top-portfolio-performance`** (`PUBLIC_CACHE_TAGS.landingTopPortfolio`).
- **Preset empty-state** (`PresetBentoGrid`) shows **total return %** only — **no $** on those cards (still uses **`RankedConfig`** for badges / return).

## Inconsistencies: expected vs problematic + fix directions

Below, **“valid”** means the product can intentionally show two different numbers. **“Problematic”** means the same *declared* metric (e.g. hypothetical $10k NAV for the same config + same close) disagrees without a user-visible reason — treat as bug or UX debt.

## Findings: five-way trace (Top 1 weekly equal, active strategy slug)

### Plain language: what “`DIAG_API_BASE` + `next dev`” and “`DIAG_PROFILE_ID`” mean

The diagnostic script runs **outside** the Next.js server, so it cannot call `mergeExplorePortfoliosEquitySeriesLiveTails` in-process (that chain needs Next’s `unstable_cache`). **`DIAG_API_BASE=http://127.0.0.1:3000`** tells the script to **HTTP `fetch` your running app** so it executes the **real** route handler for **`GET /api/platform/explore-portfolios-equity-series`**, including **merge + `livePoint`**, exactly as the browser would see after `next dev` (or any deployed URL). **`DIAG_PROFILE_ID=<uuid>`** tells the script to load **`user_portfolio_profiles`** by id and, if that profile’s `strategy_id` + `config_id` match **top1-weekly-equal**, print the **same rebased-series “last `aiPortfolio`”** math the **`user-portfolio-performance`** API uses (disk snapshot only in the script — no `ensure`).
